错误处理

# 1. 统一响应

后端提供 RESTful API 给前端时，需要响应前端 API 调用是否成功：

  • 如果成功，成功的数据是什么。后续，前端会将数据渲染到页面上

  • 如果失败，失败的原因是什么。一般，前端会将原因弹出提示给用户

因此，需要有统一响应，而不能是每个接口定义自己的风格。一般来说，统一响应返回信息如下：

  • 成功时，返回成功的状态码 + 数据

  • 失败时，返回失败的状态码 + 错误提示

在标准的 RESTful API 的定义，是推荐使用 HTTP 响应状态码 (opens new window) 作为状态码。一般来说，我们实践很少这么去做，主要原因如下：

  • 业务返回的错误状态码很多，HTTP 响应状态码无法很好的映射。例如说，活动还未开始、订单已取消等等

  • 学习成本高，开发者对 HTTP 响应状态码不是很了解。例如说，可能只知道 200、403、404、500 几种常见的

# R 和 Res<T, E>

public class R<T> {

   /**
    * 状态码
    */
   @ApiModelProperty("状态码，200为成功，非200为失败")
   protected int code;

   /**
    * 状态描述
    */
   @ApiModelProperty("信息，成功为空，失败返回错误信息")
   protected String msg;

   /**
    * 返回数据
    */
   @ApiModelProperty("数据")
   protected T data;
}


public class Res<T, K> extends R<T>{

   private K err;
}

普通业务 Api 通常我们使用 R 作为数据结构返回给前端即可。然而，当我们在数据异常，即 code 不是 200 时（数据正常为200，数据异常为非200，），data为 null时，我们需要返回和正常 data 不一致的数据结构，这时候需要用到 Res。 如下例所示：


@ApiOptions("登录")
@GetMapping("need-code")
public Res<TokenVO, Boolean> login(AccountDTO dto) {
  // 登陆成功
  TokenVO token = loginService.login()
  if(token != null){
    return Res.ok(token)
  }
  // 判断是否需要验证码
  boolean isNeedCode = loginService.isNeedCode(dto);
  return Res.error(SystemErrorCode.PARAMS_ERROR, isNeedCode);
}

# 错误码 code

错误码封装在 myapp-common-core 中，所有错误码必须实现 com.myapp.common.core.web.model.code.StatusCode接口。 在该模块下包含了全局系统的错误码。

# 错误码分类

错误码分成两类：全局的系统错误码、服务模块的业务错误码。

# 系统错误码

全局的系统错误码，使用 0-999 错误码段。

# 业务错误码

模块的业务错误码包含了3段，【服务-模块-序列号】，通过实现上述接口，应当存放在业务微服务上。如下所示:

SYS 系统，使用 1-000-000 段
USER 系统，使用 2-000-000 段
...

# 异常处理

RESTful API 发生异常时，需要拦截 Exception 异常，转换成统一响应的格式，否则前端无法处理。

# 接口的异常

在项目中，全局处理器，通过使用 Spring mvc @RestControllerAdvice 注解实现，代码如下：

/**
 * 全局异常处理器
 *
 *
 */
@Order(ExceptionOrder.GLOBAL_ORDER)
@RestControllerAdvice
public class GlobalExceptionHandler {
  	@ExceptionHandler(ApiResultException.class)
	public R apiResultException(ApiResultException e){
		return e.getRes();
	}
}

# Filter 的异常

在请求被 Spring MVC 处理之前，是先经过 Filter 处理的，此时发生异常时，是无法通过 @ExceptionHandler 注解来处理的。只能通过 try catch 的方式来实现
