文件存储

介绍

  • 什么是分布式文件

分布式文件系统是指文件系统管理的物理存储资源不一定直接连接在本地节点上，而是通过计算机网络与节点相连。

  • 为什么要使用分布式文件

分布式文件系统是面对互联网的需求而产生，互联网时代对海量数据如何存储？靠简单的增加硬盘的个数已经满足不了我们的要求，因为硬盘传输速度有限但是数据在急剧增长，另外我们还要要做好数据备份、数据安全等。

# 项目支持三类文件存储器

推荐：S3 协议的对象存储

  1. 兼容 S3 协议的对象存储：支持 MinIO、腾讯云 COS、七牛云 Kodo、华为云 OBS、亚马逊 S3 等等。 如果无法使用云服务，可以自己搭建一个 MinIO 服务。

其次：数据库存储

  1. 数据库存储：MySQL、Oracle、PostgreSQL、SQL Server 等等。

不建议：本地存储

  1. 磁盘存储：本地、FTP 服务器、SFTP 服务器。

# 文件信息获取

文件信息缓存在 Redis 中，其它微服务需要调用文件的地址信息用于显示查看，可以依赖 myapp-file-cache 模块， 通过使用 FileCacheHander.java

 /**
  * 文件缓存处理类
  */
 public class FileCacheHander {
 
     @Autowired
     private RedisService redisService;
 
 
     /**
      * 判断是否存在 该缓存 key
      * @return
      */
     public boolean keyExists(){
         return redisService.exists(getKey());
     }
 
     /**
      * 保存缓存
      * @param v
      */
     public void saveCache(FileCacheInfo v){
         redisService.hSet(getKey(),v.getId()+"",v);
     }
 
     /**
      *  删除 文件缓存
      * @param id
      */
     public void deleteCache(long id){
         redisService.hDel(getKey(),id+"");
     }
 
     /**
      * 获取缓存
      * @param id
      * @return
      */
     public FileCacheInfo getCache(long id){
         return redisService.hGet(getKey(),id+"");
     }
 
     /**
      * 获取缓存最后更新时间
      * @return
      */
     public Long getUpdateTime(){
        return redisService.get(FileRedisKey.FIEL_INFO_UPDATE_TIME);
     }
 
     /**
      * 设置最后更新缓存时间
      */
     public void setUpdateTime(){
          redisService.set(FileRedisKey.FIEL_INFO_UPDATE_TIME,System.currentTimeMillis());
     }
 
     private static IRedisKey getKey(){
         return FileRedisKey.FIEL_INFO;
     }
 }

# 文件下载

// FileController.java

   @GetMapping("/{configId}/get/{name}")
   @ApiOperation("下载文件")
   @ApiImplicitParams({
           @ApiImplicitParam(name = "configId", value = "配置编号", required = true, dataTypeClass = Long.class),
           @ApiImplicitParam(name = "name", value = "文件名称", required = true, dataTypeClass = String.class)
   })
   public void getFileContent(@PathVariable("configId") Long configId,
                              @PathVariable("name") String name, HttpServletResponse response) throws Exception {

       byte[] content = fileService.getFileContent(configId, name);
       if (content == null) {
           LOG.warn(String.format("[getFileContent][configId(%s) name(%s) 文件不存在]", configId, name));
           response.setStatus(HttpStatus.NOT_FOUND.value());
           return;
       }
       response.setContentLength(content.length);
       response.setContentType(HttpUtil.getMimeType(name, MediaType.APPLICATION_OCTET_STREAM_VALUE));
       IoUtil.write(response.getOutputStream(), false, content);
   }

# 下载方式

  • Windows平台安装包下载

可以从 https://min.io/download#/windows 下载 minio.exe 可执行的文件。

Windows下载后新建一个目录存放 minio 文件，例如 D:\minioData ，直接在 cmd 下运行 minio.exe server D:\minioData 。

启动成功以后如下图，最后红色字提示修改 access Key 和 Secret Key

  • 打开控制台

minio 提供了一个可视化的管理控制平台，安装好之后，在浏览器中输入(http://localhost:9000/ (opens new window))就可以访问了，默认的用户名和密码都是 minioadmin

  1. 文件处理类
import java.io.File;
import java.io.IOException;
import java.io.InputStream;

import org.apache.commons.io.FilenameUtils;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;

import com.conprofetech.lims.api.file.model.vo.FileVO;
import com.conprofetech.lims.common.utils.StringUtils;
import com.conprofetech.lims.framework.web.model.Res;
import com.conprofetech.lims.framework.web.model.code.SysStatusCode;
import com.github.tobato.fastdfs.domain.StorePath;
import com.github.tobato.fastdfs.proto.storage.DownloadCallback;
import com.github.tobato.fastdfs.service.FastFileStorageClient;

@Service
public class FdfsFileHandler {
	
	@Value("${fdfs.domain}")
	private String domain;

	@Autowired
	private FastFileStorageClient storageClient;

	public R<FileVO> upload(MultipartFile file) throws IOException { String originalFileName = FilenameUtils.getExtension(file.getOriginalFilename());
		if(StringUtils.isEmpty(originalFileName)) {
			return new R<>(SysStatusCode.PARAM_ERROR.getCode(),"文件后缀名不能为空");
		}
		StorePath storePath = storageClient.uploadFile(file.getInputStream(), file.getSize(),
				originalFileName, null);
		FileVO vo = new FileVO(domain, storePath.getFullPath(), domain + File.separatorChar+storePath.getFullPath(), storePath.getGroup(),file.getOriginalFilename(),file.getSize());
		return Res.ok(vo);
	}

	public InputStream download(String groupName, String path) {
		InputStream ins = storageClient.downloadFile(groupName, path, new DownloadCallback<InputStream>() {
			@Override
			public InputStream recv(InputStream ins) throws IOException {
				// 将此ins返回给上面的ins
				return ins;
			}
		});
		return ins;
	}

	public void delFile(String path) {
		storageClient.deleteFile(path);
	}
	
	public String getFullPath(String path){
		return StringUtils.isEmpty(path)? path: domain  + File.separatorChar +path;
	}

	public String getDomain() {
		return domain;
	}
}
