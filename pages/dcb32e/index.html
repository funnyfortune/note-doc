# swagger 使用

说明

在现在的开发过程中还有很大一部分公司都是以口口相传的方式来进行前后端的联调，而接口文档很大一部分都只停留在了说说而已的地步，或者写了代码再写文档。 还有一点就是文档的修改，定义好的接口并不是一成不变的，可能在开发过程中文档修改不止一次的变化，这个时候就会很难受了。

只要不是强制性要求，没人会愿意写这东西，而且在写的过程中，一个字母的错误就会导致联调时候的很大麻烦，但是通过 Swagger ，我们可以省略了这一步，而且文档出错率近乎于零， 只要你在写代码的时候，稍加几个注解，文档自动生成。

# @Api

在控制层 Controller 中使用 @Api 添加注解来描述接口信息。 @api 标记，用在类上，说明该类的作用。可以标记一个 Controller 类做为 Swagger 文档资源，使用方式如：

@Api(value = "字典数据接口文档", tags = { "字典数据相关接口" })
@RestController
@RequestMapping("/hr/dict")
public class HrDictDataController extends BaseController

# @ApiParam

@ApiParam只与参数注释一起使用( @PathParam， @QueryParam， @HeaderParam， @FormParam 和在 JAX-RS 2 中，@BeanParam)，请求属性，使用方式：

@Path("/{username}")
@ApiOperation(value = "Updated user",
   notes = "This can only be done by the logged in user.")
public Response updateUser(
     @ApiParam(value = "name that need to be updated", required = true) @PathParam("username") String username,
     @ApiParam(value = "Updated user object", required = true) User user) {...}

# API详细说明

作用范围 API 使用位置
协议集描述 @Api 用于controller类上
对象属性 @ApiModelProperty 用在出入参数对象的字段上
协议描述 @ApiOperation 用在controller的方法上
Response集 @ApiResponses 用在controller的方法上
Response @ApiResponse 用在 @ApiResponses里边
非对象参数集 @ApiImplicitParams 用在controller的方法上
非对象参数描述 @ApiImplicitParam 用在@ApiImplicitParams的方法里边
描述返回对象的意义 @ApiModel 用在返回对象类上

更详细的使用，请参考swagger官方注解文档 (opens new window)

# 微服务使用

怎么聚合各个微服务的文档？

各个微服务聚合在前端UI的增强解决方案Knife4j (opens new window)中，对比swagger相比有以下优势，通过对网关的引入Knife4j的相关依赖，对swagger的v2/api-docs进行一个拦截处理。

# 一.业务摸块

swagger整合在 myapp-common-core 模块，可以直接依赖后使用。

1、业务模块的pom.xml添加依赖

<dependency>
   <groupId>com.myapp</groupId>
   <artifactId>myapp-common-core</artifactId>
</dependency>

# 二.网关配置

注意

网关聚合了 Swagger 各个微服务的文档，提供统一访问入口。 Swagger 文档默认是禁用的，如果需要，则可以配置中心 swagger.enabled: true 进行启用，生产环境建议不要开启。 进行启用。

配置注册中心nacos中，通过配置myapp-gateway.yaml中的路由信息

  1. 打开 nacos控制台[配置管理->配置列表] ,查找myapp-gateway.yaml，点击[修改]。
  2. 添加新的微服务路由信息，设置metadata: tag: [自定义微服务api文档名称]。如下代码：
spring:
 cloud:
   gateway:
     routes:
       - filters:
           - args:
               parts: '1'
             name: StripPrefix
         id: myapp-system
         order: 1
         metadata:
           tag: 系统API   # swaager 文档分栏
         predicates:
           - args:
               pattern: /sys/**
             name: Path
         uri: 'lb://myapp-system'

访问安全

笔记

在日常开发中，接口是程序对外提供服务的重要渠道，其信息的安全性也是我们最为关注的。避免不法之人根据接口信息，对服务进行攻击。 在设计时，添加了对访问权限校验，访问只能通过管理后台，请求带上了 token 才具有访问权限。 另外，可以通过配置中心动态：启用或禁用。建议可以在开发时启用，生产时禁用。

    @Bean
    public RouterFunction<ServerResponse> indexRouter(
            @Value("classpath:/swagger/doc.html") final Resource indexHtml) {
        return route(GET("/doc.html"), request -> !enabled || StringUtils.isEmpty(request.queryParam(PARAM_NAME).get())
                ? ok().contentType(MediaType.TEXT_HTML).bodyValue("没有权限访问")
                :ok().contentType(MediaType.TEXT_HTML)
                .bodyValue(indexHtml));
    }
}
