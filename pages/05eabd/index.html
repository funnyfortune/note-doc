权限认证

介绍

  • 什么是权限认证

身份认证，就是判断一个用户是否为合法用户的处理过程。最常用的简单身份认证方式是系统通过核对用户输入的用户名和口令，看其是否与系统中存储的该用户的用户名和口令一致，来判断用户身份是否正确。

  • 为什么要使用权限认证

登录请求后台接口，为了安全认证，所有请求都携带 token 信息进行安全认证，比如使用 vue 、 react 后者 h5 开发的 app ，用于控制可访问系统的资源。

权限认证包含了myapp-auth认证授权中心和为微服务提供鉴权认证模块myapp-common-security。

平台的安全认证是基于 Spring Security 5.x 的安全框架, 支持 OAuth 2.0 认证,可与内部的传统Web应用与移动端的一些应用进行单点集成与认证授权的集成.。

# 认证授权中心

顾名思义，就是对系统登录用户的进行认证过程。 SysLoginController ` 控制器提供了【账号密码登录，短信登录，社交登录】，具体流程如下图所示:

提示

认证参与方

  • 前端应用 前端负责从从后端获取数据，在前端进行展示。
  • 网关 负责请求转发，负载均衡，限流，检查TOKEN是否合法
  • 认证服务器 负责认证用户，分发访问令牌
  • 微应用 最终的数据返回者，微应用需要负责token是否有权限访问接口
  • 用户中心 负责用户与权限的数据

一个登录的过程，其中包含了相当多验证：租户信息的验证，账号的合法性的验证，验证码的验证，其中验证码在账号累计登录错误超过5次才需要提交。 在通过一些列的验证后，验证授权中心异步提交登录日志，根据uuid生成唯一的token序列号，生成登录的过期时间，为了保障token尽量的简短，减少请求带宽。 通过uuid获取redis中的用户信息。

# 鉴权认证模块使用

提示

在平台中，微服务的认证和授权的实现方式，采用了统一接入的机制，之后系统的接入都需要遵循这套机制来实现。 平台的安全认证是基于 Spring Security 5 上实现的认证，一旦用户登录完成后，会把用户信息进行相应的存储，以方便后面的不同的微应用进行使用

# 认证注解权限控制

  • @PermitAll 不需要登录能进入该方法。

  • @PreAuthorize("@auth.hasPerm('权限')") 权限认证：必须具有指定权限才能进入该方法。

  • @PreAuthorize("@auth.hasRole('角色')") 角色认证：必须具有指定角色标识才能进入该方法

无需token访问示例

    @PermitAll
    @Validated
    @PostMapping("/login")
    public R<TokenVO> login(@Valid @RequestBody UserLoginDTO dto){

数据权限示例

// 符合sys:config:list权限要求
@PreAuthorize("@auth.hasPerm('sys:config:list')")

// 符合sys:config:add或sys:config:update权限要求即可,使用 | 拼接
@PreAuthorize("@auth.hasPerm('sys:config:add | sys:config:update')")

// 要求拥有sys:config:add 并且拥有sys:config:update权限要求,使用 & 拼接
@PreAuthorize("@auth.hasPerm('sys:config:add & sys:config:update')")

// 符合sys:config-user:update或sys:config-menu:update或sys:config-doc:update权限要求即可, 使用通配符 .*
@PreAuthorize("@auth.hasPerm('sys:config-.*:update')")

// 不符合system:user:list权限要求
@PreAuthorize("@auth.lacksPermi('sys:config:list')")
