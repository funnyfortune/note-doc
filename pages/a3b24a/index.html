快速启动

目标

使用 IDEA 工具，将项目从git拉取下来，运行起来。

# 1.准备环境

产品 描述 版本 是否必须 安装教程
IntelliJ IDEA_ (opens new window) 开发工具 >=2018 是 --
JDK_ (opens new window) java环境 jdk8以上，建议jdk8 是 --
Maven_ (opens new window) java依赖管理 >=3.6 是 --
Git_ (opens new window) 代码管理 -- 是 IDEA插件gitee安装 (opens new window)
Node (opens new window) JavaScript 运行时环境 >= 12 是 --
MySQL (opens new window) 数据库服务器 >= 5.7 是 --
Redis (opens new window) redis 数据库 >= 5.0 是 --
RocketMQ (opens new window) 消息队列，本地开发可选 4.6.1 否 --
Nacos (opens new window) 服务发现配置中心框架 2.0.3 是 部署 (opens new window)
Seata_ (opens new window) 事务框架 1.3.0 是 部署 (opens new window)
Sentinel (opens new window) 流控框架，本地开发可选 1.8.1 否 sentinel控制台 (opens new window)

# 2.拉取代码

使用 IDEA (opens new window) 的gitee插件拉取码云仓库代码。

友情提示：项目的每个模块的作用，可见 《项目结构》 文档。

# 3.安装依赖

# 安装后端依赖

在安装后端依赖前，先修改根目录下的pom.xml文件，开发环境的nacos信息

     <!-- 开发环境 -->
        <profile>
            <id>dev</id>
            <properties>
                <log.dir>logs</log.dir>
                <log.level>debug</log.level>
                <nacos.addr>localhost:8848</nacos.addr>
                <nacos.group>DEFAULT_GROUP</nacos.group>
                <nacos.username>nacos</nacos.username>
                <nacos.password>nacos</nacos.password>
            </properties>
            <activation>
                <activeByDefault>true</activeByDefault>
            </activation>
        </profile>

# 以系统微服务为例

打开nacos后台，点击菜单[配置管理-配置列表]，过滤输入myapp*

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver # 3.2.0开始支持SPI可省略此配置
    type: com.alibaba.druid.pool.DruidDataSource
    dynamic:
      seata: true
      primary: master #设置默认的数据源或者数据源组,默认值即为master
      strict: false #严格匹配数据源,默认false. true未匹配到指定数据源时抛异常,false使用默认数据源
      datasource:
        master:
          url: 地址/myapp_sys?allowMultiQueries=true&useUnicode=true&characterEncoding=UTF-8&useSSL=false
          username: 账号
          password: 密码
          # 从库数据源
      #        slave:
      #          # 从数据源开关/默认关闭
      #          enabled: false
      #          url:
      #          username:
      #          password:
      druid:
        # 初始连接数
        initialSize: 5
        # 最小连接池数量
        minIdle: 5
        # 最大连接池数量
        maxActive: 20
        # 配置获取连接等待超时的时间
        maxWait: 60000
        # 配置间隔多久才进行一次检测，检测需要关闭的空闲连接，单位是毫秒
        timeBetweenEvictionRunsMillis: 60000
        # 配置一个连接在池中最小生存的时间，单位是毫秒
        minEvictableIdleTimeMillis: 300000
        # 配置一个连接在池中最大生存的时间，单位是毫秒
        maxEvictableIdleTimeMillis: 900000
        # 配置检测连接是否有效
        validationQuery: SELECT 1 FROM DUAL
        testWhileIdle: true
        testOnBorrow: false
        testOnReturn: false
# 数据库监控
    druid:
      web-stat-filter:
        enabled: true
      stat-view-servlet:
        enabled: true
        # 设置白名单，不填则允许所有访问
        allow:
        url-pattern: /druid/*
        # 控制台管理用户名和密码
        login-username: 账号
        login-password: 密码
      filter:
        stat:
          enabled: true
          # 慢SQL记录
          log-slow-sql: true
          slow-sql-millis: 1000
          merge-sql: true
        wall:
          config:
            multi-statement-allow: true
  cloud:
    stream:
      rocketmq:
        binder:
          name-server: 地址
        bindings:
          output:
            producer:
              group: operLogGroup
          input:
            consumer:
              push:
                orderly: true
      bindings:
        output:
          content-type: application/json
          destination: operLogTopic
        input:
          content-type: application/json
          destination: operLogTopic
          group: operLogGroup

spring:
  # redis 配置
  redis:
    # 地址
    host: 127.0.0.1
    # 端口，默认为6379
    port: 6379
    # 密码
    password:
    # 连接超时时间
    timeout: 10s
    lettuce:
      pool:
        max-active: -1  # 连接池最大连接数（使用负值表示没有限制）
        max-wait: -1ms      # 连接池最大阻塞等待时间（使用负值表示没有限制）
        max-idle: 5      # 连接池中的最大空闲连接
        min-idle: 5       # 连接池中的最小空闲连接
    cache:
      type: redis
